# Карта Хайрула — PWA + сборка APK через PWABuilder

Это переписанная версия: **не нативный Kotlin, а PWA** (веб-приложение).
Его не надо собирать Gradle'ом — PWABuilder сам завернёт его в APK. Никакой Android Studio.

Внутри уже есть всё, что нужно для PWA: **манифест**, **иконки** (включая maskable),
и **service worker** для **офлайн-режима**. После первого захода приложение работает без сети.

---

## TL;DR (коротко)

1. Залить папку на любой HTTPS-хостинг (проще всего — GitHub Pages или Netlify Drop).
2. Открыть **https://www.pwabuilder.com**, вставить URL сайта.
3. Убедиться, что Manifest и Service Worker зелёные → **Package For Stores → Android**.
4. Скачать APK, поставить на телефон. Готово.

Бонус: можно вообще без APK — открыть сайт в Chrome на телефоне → «Установить приложение».

---

## 0. Почему так, а не нативно

Нативная сборка спотыкалась на Gradle/SDK (та самая боль с путями и зависимостями).
PWA убирает этот этап: логика та же (карта, метки, фильтры, прогресс, импорт/экспорт,
своя карта), но всё на вебе, а PWABuilder делает APK через TWA (Trusted Web Activity).
Минус один: при первом запуске нужен интернет, чтобы установился service worker —
дальше офлайн.

---

## 1. Локальная проверка (по желанию)

⚠️ Важно: service worker регистрируется **только** на `https://` или `http://localhost`.
Если просто открыть `index.html` двойным кликом (`file://`) — карта будет работать, но
офлайн-кэш и установка PWA — нет. Поэтому для проверки подними локальный сервер:

```bash
# из папки проекта
python -m http.server 8080
# затем открыть http://localhost:8080
```

Либо плагин **Live Server** в VS Code. На `localhost` PWA-функции включаются.

---

## 2. Хостинг (нужен HTTPS — PWABuilder читает живой URL)

### Вариант А — GitHub Pages
1. Создай репозиторий, залей **содержимое папки** (чтобы `index.html` лежал в корне
   репо или в `/docs`).
2. Settings → Pages → Source: ветка `main`, папка `/root` (или `/docs`).
3. Через минуту сайт будет на `https://ТВОЙ_НИК.github.io/ИМЯ_РЕПО/`.
   - Пути в проекте **относительные**, поэтому в подпапке `/ИМЯ_РЕПО/` всё работает.

### Вариант Б — Netlify Drop (без аккаунта, быстрее всего)
1. Открой **https://app.netlify.com/drop**.
2. Перетащи туда **папку целиком**. Получишь готовый `https://...netlify.app` URL.

(Cloudflare Pages / Vercel — тоже подойдут, логика та же.)

Проверь, что по адресу открываются и `…/manifest.webmanifest`, и `…/sw.js` — не 404.

---

## 3. Сборка APK в PWABuilder

1. Зайди на **https://www.pwabuilder.com**.
2. Вставь URL своего сайта → **Start**.
3. PWABuilder проверит PWA. Должны быть зелёными:
   - **Manifest** (имя, иконки, `display: standalone`, theme/background color — всё уже есть);
   - **Service Worker** (офлайн — уже есть).
   Если что-то жёлтое — обычно это необязательные пункты (скриншоты, доп. поля), на сборку
   Android они не влияют.
4. Жми **Package For Stores** → карточка **Android** → **Generate Package**.
5. В диалоге Android:
   - **Package ID**: например `com.goslimps.botwmap` (можно свой, формат `xxx.yyy.zzz`).
   - **App name**: `Карта Хайрула`.
   - **Signing key**: оставь **Create new** — PWABuilder сгенерит ключ и положит его в
     zip (файл `signing.keystore` + пароли в `signing-key-info.txt`). **Сохрани их** —
     понадобятся для будущих обновлений.
6. Скачается zip. Внутри: **`app-release-signed.apk`** (ставить на телефон) и
   **`app-release-bundle.aab`** (для загрузки в Google Play, если захочешь).

---

## 4. (Опционально) Убрать адресную строку — Digital Asset Links

TWA по умолчанию может показывать тонкую адресную строку Chrome. Чтобы её не было и
приложение было «полноэкранным/доверенным»:

1. В скачанном zip есть **`assetlinks.json`** (в папке рядом с инструкцией PWABuilder).
2. Залей его на хостинг по пути **`/.well-known/assetlinks.json`**
   (т.е. `https://твой-домен/.well-known/assetlinks.json`).
3. Он связывает отпечаток ключа подписи с доменом. После этого TWA открывается без строки.

На GitHub Pages папку `.well-known` тоже можно положить в корень публикации.
Если пропустить этот шаг — приложение всё равно ставится и работает, просто со строкой.

---

## 5. Установка APK на телефон

1. Перекинь `app-release-signed.apk` на телефон.
2. Открой его файловым менеджером → разреши «установку из неизвестных источников»
   для этого приложения, если попросит.
3. Установится как обычное приложение с иконкой-пином.

---

## 6. Бонус: установка как PWA без APK

На Android (Chrome): открой сайт → меню (⋮) → **«Установить приложение»** /
«Add to Home Screen». Появится иконка, запуск в отдельном окне, офлайн — всё то же,
просто без APK-файла. На iOS (Safari): «Поделиться» → «На экран Домой».

## 🗺️ Про карту и копирайт

Тайлы из игры — собственность Nintendo, в проект они **не зашиты**. По умолчанию рисуется
своя **схематичная** карта (регионы + сетка + компас). Свою картинку грузишь через
меню (⋮) → **«Загрузить свою карту»** — она ляжет в квадрат `0..1000`. Картинка хранится
локально в браузере (IndexedDB) и переживает перезапуск. Лучше всего — квадратное
изображение.

---

## 📍 Система координат

Нормализованный квадрат **0..1000**:

| ось | 0 | 1000 |
|-----|------|-------|
| `x` | запад | восток |
| `y` | **север** | **юг** |

Верхний-левый угол — северо-запад. Под свою карту просто пересчитай координаты меток в эти 0..1000.

---

## 📦 Импорт / экспорт JSON

Меню (⋮) → **«Импорт JSON»** заменяет весь набор меток, **«Экспорт JSON»** выгружает текущий.
Формат:

```json
{
  "markers": [
    {
      "id": "tower_central",
      "name": "Центральная башня",
      "category": "TOWER",
      "x": 515,
      "y": 430,
      "region": "Центральный Хайрул",
      "description": "Любой текст-подсказка"
    }
  ]
}
```

Поля: `id` (уникальный, по нему хранится «найдено»), `name`, `category` (константа из списка
ниже), `x`/`y` (0..1000), `region` и `description` — необязательные.

**Константы `category`:** `SHRINE` · `TOWER` · `KOROK` · `DIVINE_BEAST` · `GREAT_FAIRY` ·
`TECH_LAB` · `VILLAGE` · `STABLE` · `MEMORY` · `SHRINE_QUEST` · `SIDE_QUEST` · `MAIN_QUEST` ·
`TREASURE` · `MERCHANT` · `DLC` · `OTHER`.

Полные 120 святилищ / 900 короков добавляются своим JSON — структура под это сделана.
Встроенные ~60 меток (`SEED` в `app.js`) — кураторский ориентировочный набор с
приблизительными координатами, не выгрузка чужого атласа.

## 🧭 Что умеет

- Жесты: палец — пан, щипок — зум, двойной тап/двойной клик — приблизить, колесо мыши — зум.
- Тап по метке — карточка снизу (категория, регион, описание, координаты, «найдено», «в центр»).
- Поиск по имени/региону с выпадашкой → центрирует карту.
- Чипы-фильтры категорий (цвет + счётчик «найдено/всего»), общий счётчик поверх карты.
- Меню: своя карта, импорт/экспорт, вкл/выкл все категории, в центр, сброс к встроенному набору.
- Прогресс, выбранная карта и фильтры сохраняются (localStorage + IndexedDB).

## 🗂️ Структура

```
HyruleMapPWA/
├─ index.html              # каркас + стили (Material You, тёмная тема)
├─ app.js                  # вся логика: карта, метки, поиск, фильтры, импорт/экспорт
├─ manifest.webmanifest    # манифест PWA (имя, иконки, цвета, standalone)
├─ sw.js                   # service worker — офлайн-кэш
├─ icons/                  # 192/512 (any + maskable), apple-touch, favicon
│  ├─ icon-192.png  icon-512.png
│  ├─ icon-192-maskable.png  icon-512-maskable.png
│  ├─ apple-touch-icon.png  favicon-32.png
└─ ИНСТРУКЦИЯ.md           # этот файл
```

Менять данные — в `app.js` (массив `SEED`) или импортом JSON. Менять вид/цвета — в `<style>`
внутри `index.html` (переменные `--primary`, `--bg` и т.д.).
